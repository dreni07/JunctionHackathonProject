01 / Overview

A brain, not a chatbot.

Pyramid AI is a multi-agent decision system — nine specialized LangChain agents, each with exclusive tools, coordinated through a LangGraph state machine. A fusion engine merges venue fit, calendar availability, and pricing into one recommendation. The system remembers context through vector embeddings in Qdrant, keeps live sessions in Redis, and improves from real booking outcomes via reinforcement learning.

03 / AI memory · Qdrant

Memory lives in the vector DB.

Agent memory is not stored in prompts alone. Every document chunk, past conversation turn, venue fact, and pricing reference is converted to 768-dim embeddings and indexed in Qdrant. Before each agent step, LangChain retrieves the top-k nearest vectors and injects them as grounded context — so Kleopatra remembers what was said, what was uploaded, and what the Pyramid actually offers.

◆ What Qdrant stores

  • hackathon_documents — uploaded briefs, floor plans, policy PDFs (chunked + embedded)
  • session_memory:{uuid} — vectorized conversation turns per organizer session
  • venue_knowledge — space descriptions, functional types, tenant policies
  • pricing_comps — historical event pricing embedded for semantic lookup

◆ How agents use it

  • file_search tool → embed query → Qdrant similarity search → ranked passages
  • PyramidKnowledgeAgent → structured retrieval over facility vectors
  • Cross-session recall — returning users get context without re-explaining
  • Writes on every confirmed fact; TTL + upsert keeps memory fresh

07 / Reinforcement learning

Learns from its own decisions.

Confirmed bookings emit +1.0 reward signals; rejections and revisions emit negative weights. A nightly batch adjusts scoring weights, enriches the pricing dataset, and re-embeds successful outcomes into Qdrant so future retrieval surfaces proven venue + price combinations.

◆ Training sources

  • Confirmed event_requests in PostgreSQL
  • PricingReference table growth
  • Qdrant vectors from accepted proposals
  • ResearcherAgent market comps

◆ What updates

  • Preferred functional_type maps
  • Capacity score bias
  • Qdrant pricing_comps collection
  • Few-shot examples in agent prompts

◆ Guardrails

  • Only confirmed-closed events train
  • Min sample_size = 3 before shift
  • LangSmith audit replay
  • Ops config overrides live
